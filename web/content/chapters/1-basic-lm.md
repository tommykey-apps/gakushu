# 基礎LM: 次の文字を当てるゲーム

## AIって何をやってるの？

AIがやってることは「次の文字を当てるゲーム」。

「も」と入力されたら「次は "も" かな？ "た" かな？」と予測を返す。ChatGPT のような大規模なものも、根っこの仕組みはこれと同じ。

このプログラムがやることは3つだけ:

1. **予測**: 文字を入れたら、表の数値を使って「次の文字」の予測が出る
2. **学習**: 予測して → ハズレを測って → 表を直す、を300回繰り返す
3. **生成**: 学習した表を使って、予測を繰り返して文章を作る

---

## 文字を数値に変換する

コンピュータは「も」「た」みたいな文字をそのまま計算に使えない。だから最初に、文字に番号を振る。

| 文字 | う | か | た | は | も | ら | ろ | ま | れ |
|------|----|----|----|----|----|----|----|----|-----|
| 番号 | 0  | 1  | 2  | 3  | 4  | 5  | 6  | 7  | 8  |

これで「ももたろう」は `[4, 4, 2, 6, 0]` という数値の列になって、計算に使える。

---

## 予測に使う3つの表

じゃあどうやって次の文字を当てるのか？ プログラムの中に「数値がたくさん入った表」がある。入力された文字とこの表の数値を使って計算すると、予測が1つ出てくる。

| 表の名前 | 大きさ | 何をするための表か |
|----|--------|----|
| 文字の変換表 | 9文字 × 8個の数値 = 72個 | 文字番号を渡すと、その文字を8個の数値に変換して返してくれる表。辞書みたいなもの |
| 予測の点数表 | 8 × 9 = 72個 | 変換表から出てきた8個の数値と掛け合わせて、次にどの文字が来そうかの「点数」を出すための表 |
| 予測の調整値 | 9個 | 点数に足す微調整の数値 |

合計 **153個** の数値。これがこのAIの全て。

下の表は、文字の変換表 (emb\_weight) の中身。色が濃いほど値が大きい。セルにホバーすると数値が見える。学習が進むと、この数値が変わっていく。

## 予測の流れ (forward)

3つの表を紹介したところで、実際に「も」を入力したら何が起きるか見てみる。

**① 文字の変換表を引く**
文字番号を使って変換表からその文字の行を取り出す。「も」= 番号4 なら、変換表の4行目を取り出す。8個の数値が出てくる。

**② 点数表・調整値で計算する**
変換表から取り出した8個の数値と、点数表の数値を1つずつ掛けて全部足す。さらに調整値を足す。これで「次にどの文字が来そうか」の点数が9文字ぶん出てくる。

::math-sigma-explainer
::

なぜ「掛けて足す」で予測になるのか？ 点数表の中の数値が大きいところほど、結果に強く影響する。学習を繰り返すと、正解の文字に高い点数がつくように数値が調整されていく。

ただし、掛け算だけだと「この文字はそもそも来やすい/来にくい」という全体的な底上げ・底下げができない。テストで例えると、点数表が「配点 × 回答」で出した素点なら、調整値は「この科目は難しかったから全員に+5点」のような加点。

**③ 点数をパーセントに変換する**
点数のままだと「た=3.1, も=2.8」で何%くらいありそうか比べにくいし、マイナスの値もある。そこで、どんな数値でもパーセントに変換できる数学の公式を使う。

::math-exp-explainer
::

全部正の数にしてから、合計で割って100%にする。これが softmax。

## ハズレを測る (loss)

予測の仕組みはわかった。でも最初は表の数値が適当だから、予測は全然当たらない。ここからが本番で、**予測 → ハズレ測定 → 修正** を繰り返して、表の数値を良くしていく。

AIの世界では、この表の数値のことを「**重み (weight)**」、重みを何度も直していく作業のことを「**学習 (training)**」と呼ぶ。でも中身は「表の数値を書き換えてるだけ」。

予測が出たら「どのくらい間違えたか」を1つの数値にしたい。この数値が大きいほどハズレがひどく、小さいほど良い予測。

考え方はシンプルで、正解の文字に何%を振れたかを見る:
- 正解が「も」で、「も」に60%を振れていた → 良い → ハズレは小さい
- 正解が「も」で、「も」に2%しか振れてなかった → ひどい → ハズレは大きい

正解の確率が低いほどハズレの度合いが大きくなる、という計算をしたい。これにぴったりな数学の関数 (log) がある。

## 表の数値を直す (backward)

ここが学習の本体。予測確率と正解から「変換表・点数表・調整値のどの数値を、どっちの方向に、どれだけ直せばいいか」の修正指示を計算して、実際に数値を直す。

### 修正指示は予測と逆の順番で求める

予測の流れ:
```
文字 → [変換表] → [点数表] + [調整値] → 点数 → パーセント
```

修正指示はこの逆:

::viz-gradient-flow
::

**① パーセントと正解の差を出す**
「予測確率 - 正解」で、点数に対する修正指示を求める。
- 「も」が正解で、予測が「も=45%, た=20%, ...」だった場合:
  - 「も」: 0.45 - 1 = -0.55 →「あと55%ぶん上げろ」
  - 「た」: 0.20 - 0 = +0.20 →「20%ぶん下げろ」

**② 点数表への修正指示を求める**
**③ 調整値への修正指示を求める**
**④ 変換表への修正指示を求める**

修正指示が出たら、3つの表すべてに同じ式で数値を直す:
```
新しい数値 = 今の数値 - 学習率 × 修正指示
```

例: 変換表の中のある数値が今 `0.30` で、修正指示が `0.05` (下げろ) なら:
```
0.30 - 1.0 × 0.05 = 0.25
```

## 文章を生成する (generate)

300回も数値を直した表は、もうだいぶ良い予測ができるようになっている。

生成でやることは、予測を繰り返すだけ:
1. 「も」を入力 → 予測が出る (も=60%, た=20%, か=15%...)
2. この確率に従って1文字を選ぶ → 「も」が選ばれた
3. 選ばれた「も」を次の入力にする → また予測 → 「た」が選ばれた
4. ...を繰り返して「ももたろうは...」のような文章ができあがる

毎回確率に基づいてランダムに選ぶので、実行するたびにちょっと違う文章が出てくる。

## 専門用語との対応表

ここまで読めば、もう中身はわかっているはず。あとはそれぞれの操作に業界でどんな名前がついているかだけ。

| このドキュメントでの説明 | 専門用語 |
|---|---|
| 表の数値 | 重み (weight) / パラメータ (parameter) |
| 表の数値を何度も直していく作業 | 学習 (training) |
| 文字番号で変換表を引いて数値を取り出す | Embedding (埋め込み) |
| 掛け算と足し算で点数を出す | Linear (線形変換) |
| 点数をパーセントに変換する | softmax (ソフトマックス) |
| ハズレを1つの数値にしたもの | Cross Entropy Loss (交差エントロピー損失) |
| 各数値への修正指示 | 勾配 (gradient) |
| 修正指示を後ろから前に伝えること | 逆伝播 (Backpropagation) |
| 修正指示に従って数値を直すやり方 | SGD (確率的勾配降下法) |
| データを前に流して予測を出す処理 | 順伝播 (Forward pass) |
